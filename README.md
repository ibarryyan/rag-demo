## 🎯 项目简介

一个使用Go语言构建的RAG（检索增强生成）系统对比演示，通过对比纯DeepSeek回答与RAG增强回答，直观展示向量数据库如何解决大语言模型的知识过时和幻觉问题。

### ✨ 核心特性

- **🔄 双路径对比**：同时展示纯DeepSeek回答与RAG增强回答
- **🔍 向量检索**：使用Milvus实现高效的语义相似度搜索
- **📊 性能分析**：对比响应时间、准确率和信息质量
- **🚀 极简部署**：单文件实现，依赖少，易于运行
- **🧪 内置测试**：包含精心设计的测试用例集

## 📋 技术栈

| 组件 | 技术 | 用途 |
|------|------|------|
| **后端** | Go 1.21+ | 高性能服务端 |
| **向量数据库** | Milvus 2.3+ | 存储和检索文档向量 |
| **大语言模型** | DeepSeek API | 文本生成与理解 |
| **向量处理** | 自定义简化算法 | 文档向量化（演示用） |

## 🚀 快速开始

### 1. 环境要求

- Go 1.21 或更高版本
- Docker（用于运行Milvus）
- DeepSeek API密钥

### 2. 安装步骤

```bash
# 下载依赖
go mod tidy
```

### 3. 配置环境变量

创建 `.env` 文件：

```bash
# DeepSeek API密钥（必需）
DEEPSEEK_API_KEY=your_deepseek_api_key_here

# Milvus配置
MILVUS_HOST=localhost
MILVUS_PORT=19530

# 集合名称
COLLECTION_NAME=rag_demo
```

### 4. 运行程序

```bash
# 运行RAG演示
go run main.go
```

## 📈 RAG优势展示

通过本演示，您可以清晰看到RAG相比纯LLM的优势：

| 场景 | 纯DeepSeek | RAG增强 | 优势 |
|------|------------|---------|------|
| 最新信息查询 | 可能过时 | 基于最新文档 | ✅ 知识新鲜度 |
| 具体数值问题 | 可能不准确 | 精确文档引用 | ✅ 事实准确性 |
| 专业概念解释 | 基于训练数据 | 基于专业文档 | ✅ 内容可靠性 |
| 回答可验证性 | 无法验证 | 提供参考来源 | ✅ 可追溯性 |
